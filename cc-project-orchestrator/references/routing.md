# Routing Notes

## Choose Superpowers-first when
- the project idea is still fuzzy
- you need option generation, trade-offs, or MVP slicing
- the current phase is a design decision, not execution

## Choose GSD-first when
- the project already has direction and needs stateful phase control
- you are resuming an ongoing project
- you need roadmap/phase continuity more than ideation

## Use both when
- you want brainstorming, then a compact project skeleton
- you want phase context, then a concrete execution plan
- you want a review/debug pass, then phase verification

## Never do
- discover, discuss, and execute with full docs all at once
- let both suites create competing long-form specs
- create reports, retros, and summaries unless they will be read again soon
